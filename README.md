# Portfolio Project

Static portfolio site for mgriffin designs. Design tokens and layout rules live in [`DESIGN.md`](DESIGN.md). Reference pages: [`index.html`](index.html) (homepage) and [`stockandstem/index.html`](stockandstem/index.html) (case study).

## Adding a New Project

### 1. Create the project folder

Create a folder at the repo root using a URL-friendly slug (lowercase, hyphens, no spaces). Examples:

| Project title   | Folder slug              |
|-----------------|--------------------------|
| Stock & Stem    | `stockandstem`           |
| Glass Botanist  | `plant-glass-botanist`   |
| Bibliocat       | `bibliocat`              |

Folder structure:

```
{slug}/
├── content.md      ← you write this first
├── assets/         ← cover, hero, and section images
└── index.html      ← generated by Cursor (do not write by hand)
```

### 2. Add images to `assets/`

| Asset        | Purpose                                      | Notes                          |
|--------------|----------------------------------------------|--------------------------------|
| Cover image  | Homepage project card                        | 16:9 aspect ratio              |
| Hero image   | Large mockup below the title on the case study | 16:9; optional if cover suffices |
| Section images | Process galleries, solution screenshots    | One file per visual            |

Use descriptive filenames (e.g. `cover-glass-botanist.svg`, `Planner-dashboard.png`). Paths in `content.md` are relative to the project folder (`assets/...`).

### 3. Fill in `content.md`

Create `{slug}/content.md` with YAML frontmatter and Markdown body sections. **Fill in all fields before generating the page** — Cursor will stop and ask you to complete this if anything is still `[TODO]` or placeholder text.

**Required frontmatter:**

```yaml
---
title: "Project Title"
slug: project-slug
tagline: "Short subtitle under the page title."
cardDescription: "One-line blurb for the homepage card (≈120 characters max)."
coverImage: "assets/cover-project.svg"
coverAlt: "Describe the cover image."
summary: "1–2 sentences shown in the project hero."
role: "UX Designer & Developer"
timeline: "8 weeks · 2026"
tools: "Figma, HTML/CSS, Cursor"
team: "Solo independent project"
---
```

**Optional frontmatter:**

```yaml
heroImage: "assets/mockup-hero.svg"
heroAlt: "Describe the hero mockup."
externalUrl: "https://your-app.netlify.app/"
```

**Body sections** (standard order — delete any you do not need):

| Section      | What to write                                      |
|--------------|----------------------------------------------------|
| Overview     | Optional; skip if the hero summary is enough       |
| Challenge    | Problem, users affected, constraints               |
| Process      | Research, ideation, iterations; use `###` subheadings for image groups |
| Solution     | What you built; use `###` subheadings for features |
| Deployment   | Launch details and impact (or use `## Outcome`)    |
| Reflection   | Learnings and next steps                           |

**Theme comments** (optional HTML comments after a `##` heading — control colored panels and galleries):

```markdown
## Challenge
<!-- theme: lavender -->

## Process
<!-- theme: night -->
<!-- galleries: true -->

## Solution
<!-- theme: lavender -->
<!-- features: true -->
```

See [`stockandstem/content.md`](stockandstem/content.md) and [`stockandstem/index.html`](stockandstem/index.html) for a filled example.

### 4. Ask Cursor to generate the page

In Cursor chat, say something like:

> Use the **portfolio-project-page** skill to create a project page for **{Project Title}**.

Or:

> Generate `{slug}/index.html` from `{slug}/content.md` and add a homepage card.

Cursor will:

1. Validate `content.md`
2. Generate `{slug}/index.html` following `DESIGN.md`
3. Add or update a card in the homepage `#work` grid in `index.html`

**Two-step workflow:** If `content.md` is empty or incomplete, Cursor creates a template and stops. Fill it in, then ask again to generate.

### 5. Verify locally

Open `index.html` in a browser (file:// is supported). Check:

- [ ] Homepage card image and link work (`{slug}/index.html`)
- [ ] Case study nav, hero, sections, and back link render correctly
- [ ] All image paths resolve
- [ ] External app link opens (if `externalUrl` is set)

## Finding the Cursor Skill

The **portfolio-project-page** skill tells Cursor how to turn `content.md` into HTML and wire up the homepage card.

| Location | Path |
|----------|------|
| User skills (this machine) | `~/.cursor/skills/portfolio-project-page/` |
| Skill entry point | `~/.cursor/skills/portfolio-project-page/SKILL.md` |

**If you change Cursor accounts or machines**, skills do not sync automatically. To recover:

1. Check `~/.cursor/skills/portfolio-project-page/` on any machine where you set it up.
2. Copy the entire `portfolio-project-page` folder to `~/.cursor/skills/` on the new machine.
3. Or recreate it with Cursor: **create a new skill** named `portfolio-project-page` and point it at this repo's `DESIGN.md`, `stockandstem/index.html`, and the workflow in this README.

Skill support files (templates, reference):

```
~/.cursor/skills/portfolio-project-page/
├── SKILL.md
├── reference.md
├── examples.md
└── templates/
    ├── content.md
    ├── project-page.html
    └── homepage-card.html
```

You can also back up that folder in this repo or a private gist so you never lose it.

## Manual fallback (no skill)

If the skill is unavailable, you can still add a project by hand:

1. Copy `stockandstem/index.html` → `{slug}/index.html`
2. Replace copy, meta fields, images, and section content from `content.md`
3. Add a card to `#work .grid` in root `index.html` (match the Stock & Stem card pattern; link to `{slug}/index.html`)

Always follow [`DESIGN.md`](DESIGN.md) for tokens, typography, buttons, and layout.
